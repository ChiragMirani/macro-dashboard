# MacroForecastbyCM

Live site: [chiragmirani.github.io/macro-dashboard](https://chiragmirani.github.io/macro-dashboard/)
Track record: [chiragmirani.github.io/macro-dashboard/track-record.html](https://chiragmirani.github.io/macro-dashboard/track-record.html)
Maintained by: Chirag Mirani

A live, auto-updating release schedule for the most market-moving US macroeconomic prints, paired with house model forecasts and live Kalshi market consensus for direct comparison. Built and maintained solo.

## What it is

A daily-refreshed dashboard that answers, for every upcoming US macro release:

1. When does it release (day, date, time ET)?
2. What does my house model expect, built from public BLS, FRED, and Kalshi data?
3. What does the Kalshi prediction market expect, with a link to the underlying market?
4. What was the last actual print, with a link to the official source (BLS, BEA, DOL, ADP)?
5. Surprise risk: does my model think this print will deviate materially from consensus?

After each release lands, the [track record page](https://chiragmirani.github.io/macro-dashboard/track-record.html) scores the house forecast and Kalshi consensus against the actual, settled from FRED. No historical backfill. Snapshots start the day they're captured, so the record is honest.

## How the forecasts are produced

House forecasts are generated by a proprietary stack combining modern machine learning, ensemble modeling, and alternative-data signals across rates, labor, and inflation. The system is engineered for institutional rigor with strict out-of-sample discipline.

Capabilities at a glance:

1. AI and ML-driven forecasts across the full US macro release calendar
2. Ensembled point and probabilistic outputs with surprise-risk scoring
3. Cross-asset and cross-release feature engineering
4. Continuous retraining on the latest data
5. Live calibration against prediction-market consensus

Model code, training data, methodology, and live backtest results are not published. Reach out for details, demos, or consultation inquiries.

## How the data flow works

The local Windows scheduled task runs daily at 6 PM ET. A dependency-aware orchestrator reruns only the models whose FRED inputs have moved (so NFP rebuilds when new claims data arrives), then publishes the JSON outputs to this public repo.

GitHub Actions runs twice a day on weekdays. It pulls live Kalshi consensus, refreshes FRED last-release values, re-renders the static site, and commits docs.

The static site at GitHub Pages serves the rendered dashboard and a machine-readable feed at [`dashboard_data.json`](https://chiragmirani.github.io/macro-dashboard/dashboard_data.json).

## Machine-readable feeds

1. [`dashboard_data.json`](https://chiragmirani.github.io/macro-dashboard/dashboard_data.json) is the current release schedule plus forecasts.
2. [`track_record.json`](https://chiragmirani.github.io/macro-dashboard/track_record.json) is the historical scoreboard, settled vs actual.
3. [`llms.txt`](https://chiragmirani.github.io/macro-dashboard/llms.txt) is an AI-assistant friendly summary.
